# TRICK ENGINE GAME SPECIFICATION

# GLITCH

## Version 1.0

### Classification

Epistemic Game Type:
Cognitive Bias Game

Primary Psychological Focus:
Misunderstanding

Core Question:

> How often do our intuitions produce incorrect conclusions?

---

# Design Philosophy

GLITCH is designed to create failure.

Players encounter statements that feel true but are false.

The objective is not merely to be correct.

The objective is to discover how and why mistakes occur.

---

# Psychological Model

GLITCH trains:

* bias recognition
* metacognition
* skepticism
* cognitive flexibility

The central failure mode is:

> Trusting intuition without verification.

---

# Deck Characteristics

Statements should exploit:

* common misconceptions
* cognitive shortcuts
* intuitive reasoning errors

Example cards:

* Heavier objects fall faster than lighter objects.
* Memory works like a video recording.
* Humans are naturally good lie detectors.

---

# Core Mechanic

Players must declare confidence.

Confidence levels:

* Low
* Medium
* High

Points scale with confidence.

---

# Scoring

Correct + High Confidence:
+4

Correct + Medium Confidence:
+2

Correct + Low Confidence:
+1

Incorrect + High Confidence:
-4

Incorrect + Medium Confidence:
-2

Incorrect + Low Confidence:
-1

---

# Misunderstanding Score

Each incorrect answer adds:

+1 Misunderstanding Marker

Winner:

Highest score with the fewest misunderstanding markers.

---

# What It Models

GLITCH emphasizes distortion points within:

R → B → U → I → L

The game studies where reality diverges from belief.

---

# Ideal Outcome

Players become more sensitive to:

* assumptions
* intuition traps
* cognitive biases
* overconfidence

and learn to separate:

"I think this is true"

from

"I know this is true."

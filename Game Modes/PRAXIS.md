# TRICK ENGINE GAME SPECIFICATION

# PRAXIS

## Version 1.0

---

# Classification

Epistemic Game Type:
Behavioral Reality Game

Primary Psychological Focus:
Action, Consequence, and Reality Change

Core Question:

> What happens when beliefs are forced to become actions?

---

# Design Philosophy

PRAXIS is the first TRICK game that completes the full epistemic loop:

```text
R → B → U → I → L → A → R'
```

Unlike earlier games:

* ANCHOR evaluates truth
* ECHO evaluates belief spread
* GLITCH evaluates cognitive error
* MIRROR evaluates interpretation
* SPELLS evaluates influence

PRAXIS evaluates something more fundamental:

> What beliefs *do* to the world when acted upon.

It is not a communication game.

It is a consequence game.

---

# Psychological Model

PRAXIS models:

* decision-making under belief
* behavioral commitment
* consequence feedback loops
* risk under uncertainty
* belief-driven environment change

The central failure mode is:

> Believing that thinking and acting are separate systems.

---

# Core Concept

A belief is only meaningful if it can produce action.

A statement is only complete when it becomes behavior.

A communication event is only resolved when its consequences are observed.

---

# State Model

Each game event is represented as:

```text
(R, B, U, I, L, A, R')
```

Where:

| Variable | Meaning           |
| -------- | ----------------- |
| R        | Initial reality   |
| B        | Speaker belief    |
| U        | Utterance         |
| I        | Interpretation    |
| L        | Listener belief   |
| A        | Action            |
| R'       | Resulting reality |

---

# Core Mechanic

PRAXIS operates in three phases:

---

## Phase 1: Belief Formation

A statement is presented.

Players form beliefs based on interpretation.

No action yet occurs.

---

## Phase 2: Commitment

Each player chooses an action based on belief:

* Act
* Do not act
* Act opposite of belief (risk action)

Actions must be explicit.

---

## Phase 3: Reality Update

Actions are resolved.

The system computes:

```text
R' = f(R, A)
```

Reality is updated based on collective actions.

---

# Key Distinction

Unlike other TRICK games:

| Layer  | Focus                                 |
| ------ | ------------------------------------- |
| ANCHOR | Is it true?                           |
| ECHO   | What do others believe?               |
| GLITCH | Why am I wrong?                       |
| MIRROR | What did you mean?                    |
| SPELLS | What does this make people do?        |
| PRAXIS | What *actually happens* after action? |

---

# Scoring System

---

## 1. Belief Accuracy Score (BAS)

Measures alignment between belief and reality:

```text
BAS = A(L, R)
```

* correct belief → +2
* incorrect belief → -2

---

## 2. Action Correctness Score (ACS)

Measures whether action was appropriate given reality:

```text
ACS = A(A, R)
```

* beneficial action → +3
* harmful action → -3

---

## 3. Belief-Action Coherence (BAC)

Measures whether action matches belief:

```text
BAC = A(L, A)
```

* consistent → +1
* inconsistent → -1

---

## 4. Reality Impact Score (RIS)

Measures how actions change reality:

```text
RIS = A(R, R')
```

Interpretation:

* 1 → no change
* 0 → change occurred

(Scored contextually in gameplay)

---

## 5. Total Score

```text
S = BAS + ACS + BAC + RIS
```

---

# Misunderstanding Metric

PRAXIS extends misunderstanding beyond cognition into behavior:

```text
M = mismatch in (R, B, U, I, L, A)
```

High M indicates:

* wrong belief
* wrong action
* or correct belief with harmful behavior

---

# Game Modes

---

## 1. Individual PRAXIS

Focus:
Personal decision-making under uncertainty

Goal:
Maximize correct action outcomes

---

## 2. Social PRAXIS

Focus:
Group behavior and collective action

Goal:
Observe emergent coordination or collapse

---

## 3. Crisis PRAXIS

Focus:
High-stakes environments

Example scenarios:

* fire alarm
* financial panic
* misinformation outbreak

Goal:
Test robustness of belief-action systems

---

## 4. Experimental PRAXIS

Focus:
Counterfactual actions

Players may:

* simulate alternate actions
* compare outcomes
* test decision branching

---

## 5. Adversarial PRAXIS

Focus:
Competing actions under shared reality

Players may:

* act on conflicting beliefs
* influence each other’s actions indirectly
* reshape shared environment

---

# System Dynamics Insight

PRAXIS introduces a critical transformation:

### Before PRAXIS:

> Beliefs describe reality

### In PRAXIS:

> Beliefs modify reality

This turns TRICK into a closed-loop system:

```text
R₀ → B → U → I → L → A → R₁ → ...
```

---

# Key Design Principle

PRAXIS assumes:

> The ultimate test of belief is not correctness, but consequence.

---

# Ideal Outcome

Players develop:

* stronger decision calibration
* awareness of action consequences
* reduced belief-action inconsistency
* improved judgment under uncertainty

---

# Closing Principle

If ANCHOR asks:

> “What is true?”

And ECHO asks:

> “What do people believe?”

Then PRAXIS asks:

> “What happens because you believed it?”
